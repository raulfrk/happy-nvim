# Audit batch fix — todos #19–#29

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:systematic-debugging`. Each fix has its own Phase 1–4 trace below; one commit per todo.

## Context

Exploratory bug audit (prior session) surfaced 11 latent bugs in the
idle watcher, remote helpers, and Claude send path. After ultrathink
verification, 10 hold up + one (#20) has a corrected trigger:

- #19 HIGH — shell injection via user input in `remote/grep.lua` +
  `remote/browse.lua` (single-quote escape broken).
- #20 MED — `idle.mark_busy` doesn't block the next idle flip; if
  claude doesn't produce output in response to a send, watcher
  fires a false "idle" alert after DEBOUNCE_SECS (cooldown may not
  cover it across long sessions).
- #21 MED — `idle.watch_all` leaks libuv timers when the module is
  re-require'd (fresh `local timer` = nil, re-entry guard misses).
- #22 LOW — `d.lnum + (d.lnum == 0 and 0 or 0)` no-op ternary in
  `claude._build_ce_payload`; diagnostics sent to Claude show 0-based
  line numbers.
- #23 MED — `remote.dirs.pick_for_host` sends raw `cd <path>` via tmux
  send-keys; paths with spaces/quotes break.
- #24 LOW — orphan entries in `idle.states` when sessions are killed
  from another tmux client.
- #25 LOW — `idle` require silently skipped if module missing;
  mark_busy contract broken in minimal configs.
- #26 LOW — `<leader>cf` on nameless buffer sends `@.` to Claude.
- #27 LOW — `apply_flip` swallows tmux + notify-send exit codes.
- #28 LOW — `resolve_target` race with pane kill; cryptic tmux error.
- #29 LOW — unknown diag severity silently degrades to `'UNKNOWN'`.

Dropped during ultrathink: the audit's original claim that
`remote.util.run` has a concurrency bug — each invocation has its own
closure locals, no interleaving risk.
